# Dev 카테고리 GitBook 스타일 문서 위키 설계

## 목표
Dev 카테고리를 GitBook 스타일의 문서 위키로 전환하여 기술 문서를 체계적으로 정리하고 탐색할 수 있도록 한다.

## 구조 설계

### 1. 하위 카테고리 구조
```
Dev/
├── VHDL/
│   ├── 오류 분석/
│   │   ├── [중요도/빈도수 기준 정렬]
│   │   └── 오류 유형별 분류
│   ├── 문법 레퍼런스/
│   │   ├── 기본 구조
│   │   ├── 타입과 신호
│   │   └── 고급 기능
│   ├── Vivado 툴/
│   │   ├── DRC 오류 해결
│   │   ├── 합성 가이드
│   │   └── 시뮬레이션
│   └── 설계 가이드/
│       └── 프로젝트별 설계 문서
├── [향후 추가 가능]
│   ├── Python/
│   ├── JavaScript/
│   └── 기타 언어/툴
```

### 2. 페이지 레이아웃

#### 메인 Dev 페이지 (`/dev/`)
- **사이드바**: 하위 카테고리 네비게이션
  - VHDL (현재)
  - [향후 추가 가능한 다른 언어/툴]
- **메인 콘텐츠**: 
  - 각 하위 카테고리별 요약 카드
  - 최근 업데이트된 문서
  - 인기 문서 (조회수 기준)

#### 하위 카테고리 페이지 (`/dev/vhdl/`)
- **사이드바**: VHDL 문서 네비게이션
  - 오류 분석
  - 문법 레퍼런스
  - Vivado 툴
  - 설계 가이드
- **메인 콘텐츠**: 
  - 섹션별로 문서 목록
  - 중요도/빈도수 기준 정렬
  - 검색 기능 (향후)

### 3. 문서 분류 기준

#### 오류 분석
- **태그 기반**: `[오류]` 제목 또는 특정 오류 태그
- **정렬 기준**: 
  1. 중요도 (심각도)
  2. 빈도수 (조회수)
  3. 최근 발생 (날짜)

#### 문법 레퍼런스
- **태그 기반**: `문법`, `레퍼런스`, `VHDL-2008` 등
- **시리즈 기반**: `series: "VHDL 문법 정리"` 활용
- **정렬**: 시리즈 순서 또는 주제별

#### 툴 사용법
- **태그 기반**: `Vivado`, `DRC`, `Synthesis` 등
- **정렬**: 툴별, 기능별

#### 설계 가이드
- **태그 기반**: `설계`, `FPGA`, `아키텍처` 등
- **정렬**: 프로젝트별, 날짜순

### 4. UI/UX 설계

#### GitBook 스타일 특징
- **사이드바 네비게이션**: 고정 또는 접을 수 있는 사이드바
- **깔끔한 타이포그래피**: 가독성 중심
- **계층적 구조**: 명확한 섹션 구분
- **검색 기능**: 빠른 문서 찾기 (향후)
- **다크 모드 지원**: 기존 테마와 통합

#### 반응형 디자인
- **데스크톱**: 사이드바 + 메인 콘텐츠 2단 레이아웃
- **태블릿**: 접을 수 있는 사이드바
- **모바일**: 하단 네비게이션 또는 햄버거 메뉴

### 5. 데이터 구조

#### Front Matter 활용
```yaml
category: dev
subcategory: "VHDL"
tags: [VHDL, 오류, Synthesis]
series: "VHDL 문법 정리"  # 시리즈 연결
series_order: 1  # 시리즈 내 순서
```

#### 분류 로직
- **오류 분석**: 제목에 `[오류]` 포함 또는 특정 오류 태그
- **문법 레퍼런스**: `문법`, `레퍼런스` 태그 또는 `series` 필드
- **툴 사용법**: `Vivado`, `DRC` 등 툴 관련 태그
- **설계 가이드**: `설계`, `FPGA`, `아키텍처` 태그

### 6. 구현 단계

1. **Phase 1**: 기본 레이아웃 및 HTML 구조
   - dev.html 재구성
   - 사이드바 네비게이션
   - 섹션별 문서 분류

2. **Phase 2**: CSS 스타일링
   - GitBook 스타일 CSS
   - 반응형 디자인
   - 다크 모드 지원

3. **Phase 3**: 고급 기능 (향후)
   - 검색 기능
   - 문서 간 링크
   - 목차 자동 생성

### 7. 확장성

- **다른 언어/툴 추가**: subcategory만 추가하면 자동으로 사이드바에 표시
- **새로운 문서 유형**: 태그 기반으로 자동 분류
- **다국어 지원**: 향후 확장 가능
