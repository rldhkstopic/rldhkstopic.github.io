# 버전 번호 증가 규칙

현재 버전: **v0.0.63**

## 버전 형식
- **vMAJOR.MINOR.PATCH** (예: v0.0.63)
  - **MAJOR (첫 번째 자리수)**: 호환성 파괴 변경, 레이아웃/URL/빌드 파이프라인 변경
  - **MINOR (두 번째 자리수)**: 새 기능 추가 (호환성 유지)
  - **PATCH (세 번째 자리수)**: 버그 수정, 스타일 수정, 문서 보강

## 다음 버전 증가 시점

### PATCH (0.0.63 → 0.0.64)
- 버그 수정
- 스타일 변경
- 문서 보강
- 사용자 화면/동작이 바뀌는 변경

### MINOR (0.0.63 → 0.1.0)
- **새 기능 추가** (호환성 유지)
- 예: 새로운 카테고리 추가, 새로운 페이지 추가, 새로운 자동화 기능 추가

### MAJOR (0.0.63 → 1.0.0)
- **호환성 파괴 변경**
- URL 구조 변경
- 카테고리 구조 변경
- 자동화 파이프라인의 출력 형태 변경
- 레이아웃 대규모 변경

## 현재 상황

현재 **v0.0.63**이므로:
- 다음 커밋은 보통 **v0.0.64** (PATCH 증가)
- 새 기능을 추가하면 **v0.1.0** (MINOR 증가)
- Breaking change가 있으면 **v1.0.0** (MAJOR 증가)

**참고**: VERSION 파일이 v0.0.24로 되어 있지만, 실제 커밋 메시지에는 v0.0.63을 사용하고 있습니다. VERSION 파일을 최신 버전으로 업데이트하는 것을 권장합니다.
