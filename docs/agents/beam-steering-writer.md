# 디지털 빔조향 프로젝트 글쓰기 에이전트

이 문서는 **디지털 빔조향** 프로젝트 문서를 한 편의 마스터 문서에서 단계적으로 구체화할 때 따를 워크플로를 정의한다.  
에이전트(또는 작성자)는 **목차 수준3 확립 → 글쓰기 단계** 순서를 반드시 따른다.

---

## 0. 글쓰기 방식: 질문 유도식 서술

- **하지 말 것**: “이런 개념이 있고, 이런 것들이 있고, 이런 문제점이 있다” 식의 나열.
- **할 것**: **질문을 던지고 → 그에 답하면서 다음 단계로 넘어가는 방식**으로 서술한다.
- 각 섹션은 하나의 질문(또는 작은 질문들)으로 시작하고, 답을 채우면서 **다음에 자연스럽게 이어질 질문**을 남긴다.  
  예: “디지털 빔조향이란?” → 답 → “그렇다면 우리 시스템에서는 어떻게 구현되는가?” → 답 → “보정은 왜 필요한가?” → …
- 이렇게 **한 가지를 짚고 파고든 뒤 다음으로 넘어가는** 방식을 **질문 유도식(question-driven)** 또는 **단계적 질문 유도**라고 한다.  
  마스터 문서의 목차와 본문은 이 흐름에 맞춘다.

## 2. 워크플로 (필수 순서)

### 2.1 단계 1: 목차 확립 (수준 3까지)

- **목표**: 글쓰기에 들어가기 전에 **h1 → h2 → h3** 구조를 먼저 확정한다.
- **작업**:
  1. `docs/beam-steering-project.md`를 연다.
  2. 문서 제목(h1)과 대목차(h2)를 정한다.
  3. 각 h2 아래에 **h3 소목차**를 모두 정한다.  
     - 빈 섹션이어도 제목만 있어야 한다.  
     - “나중에 쓸 예정”이라도 h3 제목은 반드시 적어 둔다.
- **완료 조건**:  
  - 목차만 읽어도 문서의 흐름이 파악될 정도로 h1/h2/h3가 채워져 있어야 한다.  
  - 이 단계가 끝나기 전에는 본문 문단을 채우지 않는다.

### 2.2 단계 2: 글쓰기

- **시작 조건**: 단계 1(목차 수준3 확립)이 완료된 후에만 진행한다.
- **작업**:
  1. 목차 순서(h1 → h2 → h3)대로 **위에서부터** 각 섹션의 본문을 작성한다.
  2. **질문 유도식**으로 쓴다: 섹션마다 질문을 제시하고, 답을 채운 뒤 **다음 섹션으로 이어지는 질문**을 자연스럽게 남긴다. 한 가지를 짚고 파고든 뒤 다음으로 넘어간다.
  3. 한 번에 전체를 쓰지 않고, **섹션 단위로 구체화**해 나가도 된다.
  4. 수식은 `$...$`(인라인), `$$...$$`(디스플레이)만 사용한다.  
     - 작성 스타일은 `docs/GUIDE_WRITING_STYLE.md`를 따른다.
- **발행**:  
  - 마스터 문서가 충분히 정리되면, 필요한 구간을 잘라 `_posts/project/YYYY-MM-DD-제목.md`로 옮기고  
    `category: project`, `subcategory: 디지털 빔조향`을 넣어 블로그에 올린다.

## 5. 대상 시스템 구성 (참고용)

아래는 마스터 문서가 다루는 **실제 시스템**의 개요다. 글쓰기 시 이 구성을 기준으로 “디지털 빔조향이 무엇인지”, “이 제어를 위해 무엇이 필요한지”를 질문 유도식으로 풀어 나간다.

| 구분 | 내용 |
|------|------|
| **안테나** | 배열 안테나 10채널 중 **8채널** 사용. 8채널은 RF 케이블로 고주파 모듈과 연결. |
| **고주파 모듈** | 8채널 입력 → 각각 **LNA 모듈** → **디바이더** → **빔조향 경로 #1, #2** (각 경로당 8채널). |
| **경로 내 신호 처리** | 각 경로에 **MFC1×8**(채널 보정), **MFC2×8**(각도 보정). MFC1에서 ATT/TTD로 채널 보정 후 값이 MFC2로 전달되어 각도 보정(ATT/TTD). 8채널 합산 → **경로당 출력 1개**. 고주파 모듈 출력 = **총 2개**(경로별 1개). |
| **제어** | **제어 모듈**이 고주파 모듈 제어: LNA·MFC 제어값(경로별·채널별 **총 16개**). |
| **메모리** | 제어 모듈 내 **EEPROM** + **BRAM**. 전원 인가 시 EEPROM → BRAM LUT 로드(EEPROM 8비트, TTD+ATT 15비트 저장 불가 대응). 메모리 종류: **채널보정메모리**, **각도보정메모리**, **빔조향메모리**(조향각별 이론 TTD, 이론값으로 형성 가능). |
| **보정 데이터** | 채널 보정: VNA만으로 가능하다고 가정. **각도 보정: 근접전계에서 수집해야 하는지** (검토 필요). |
| **외부 입력** | 제어 모듈 입력: **주파수**, **조향각**, **빔조향 경로 선택**. 이에 따라 메모리 참조 후 MFC 제어. |
| **열린 질문** | 각도 보정은 근접전계가 필요한가? 구성은 갖춰졌는데 **시험은 어떻게 할 것인가?** |
