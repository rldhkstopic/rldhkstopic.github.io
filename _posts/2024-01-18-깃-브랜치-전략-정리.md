프로젝트를 진행하면서 여러 브랜치 전략을 사용해봤는데, 각각의 장단점을 정리해봤다.

## Git Flow

가장 널리 사용되는 전략이다. main, develop, feature, release, hotfix 브랜치를 사용한다.

장점:
- 명확한 역할 분담
- 대규모 프로젝트에 적합
- 배포 관리가 체계적

단점:
- 브랜치가 많아 복잡할 수 있음
- 작은 프로젝트에는 과할 수 있음

## GitHub Flow

간단하고 실용적인 전략이다. main과 feature 브랜치만 사용한다.

장점:
- 단순하고 이해하기 쉬움
- 빠른 배포 가능
- CI/CD와 잘 맞음

단점:
- 복잡한 릴리즈 관리에는 부적합

팀 규모와 프로젝트 특성에 맞는 전략을 선택하는 것이 중요하다.
