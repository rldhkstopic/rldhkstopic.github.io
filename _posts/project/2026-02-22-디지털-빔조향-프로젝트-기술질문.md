# 기술질문 (Technical Q&A)

이 문서는 BeamSteering 프로젝트 진행 과정에서 발생하는 주요 기술적 질문과 그에 대한 답변을 정리하는 공간입니다.

**Q1. 각도보정메모리 주소 할당 방식에서 각도(Angle)에 대한 내용이 직접적으로 포함되지 않은 이유는 무엇인가요? 빔조향메모리에서 출력된 TTD 값을 주소 범위에 넣기 때문에 상관없는 것인가요?**

**A1. 네, 정확합니다. 각도 정보는 '이론적 TTD 값'에 이미 내포되어 있기 때문에 직접 주소에 포함될 필요가 없습니다.**

**Q6. 각도보정메모리 주소 방식에서 서로 다른 조향각이 동일한 이론적 TTD 값을 출력하여 주소가 겹치는 문제는 발생하지 않나요? 실제 메모리에는 데이터가 어떻게 배치되나요?**

**A6. 주소 충돌은 발생하지 않으며, 동일한 TTD 값이 동일한 주소를 가리키는 것은 '의도된 설계'입니다.**

1. **주소의 고유성 (Address Uniqueness)**: 주소는 `Channel(3bit) | Frequency(7bit) | TTD(8bit)`를 비트 결합(Concatenation)하여 생성됩니다. 특정 채널의 특정 주파수 내에서 TTD 값(0~255)은 항상 고유한 오프셋을 가지므로 상위 인덱스가 다른 데이터와 겹칠 수 없습니다.
2. **물리적 보정의 원리**: 서로 다른 각도가 동일한 이론적 TTD 값(예: 128)을 가질 수 있습니다. 하지만 보정의 목적은 "소자가 TTD=128로 설정되었을 때 발생하는 물리적 오차를 잡는 것"입니다. 설정값이 같다면 오차도 동일하므로 같은 보정 데이터를 참조하는 것이 정확합니다.
3. **메모리 배치 (Memory Layout)**: 메모리는 `Memory[Channel][Frequency][TTD_Value]` 형태의 3차원 배열 구조로 관리됩니다. 채널당 구역을 나누고, 그 안에서 주파수별로 256개의 TTD 슬롯을 배치하여 하드웨어가 즉각적으로 데이터를 찾아갈 수 있도록 설계되었습니다.
