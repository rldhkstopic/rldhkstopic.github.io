# 블로그 포스트 작성 가이드

AI와의 대화 내용을 블로그 포스트로 변환하기 위한 프롬프트 가이드입니다.

## 역할 지정

다음 역할을 명확히 지정하여 AI에게 요청하세요:

### 역할 1: 기술 문서 작성자
```
당신은 기술 블로그 작가입니다. AI와의 대화 내용을 바탕으로 
명확하고 읽기 쉬운 기술 문서를 작성해주세요.
```

### 역할 2: 학습 내용 정리자
```
당신은 학습 내용을 체계적으로 정리하는 전문가입니다. 
대화 내용에서 핵심 개념, 예제, 해결 과정을 추출하여 
학습자가 이해하기 쉽게 구조화해주세요.
```

### 역할 3: 문제 해결 과정 기록자
```
당신은 문제 해결 과정을 상세히 기록하는 기술 문서 작가입니다.
문제 상황, 시도한 방법, 최종 해결책을 명확히 구분하여 작성해주세요.
```

## 프롬프트 템플릿

### 기본 템플릿
```
위 대화 내용을 바탕으로 블로그 포스트를 작성해주세요.

요구사항:
1. 제목: [주제에 맞는 명확한 제목]
2. 카테고리: [study/dev/daily/document 중 선택]
3. 구조:
   - 서론: 문제 상황 또는 학습 목표
   - 본문: 핵심 내용, 해결 과정, 예제 코드
   - 결론: 요약 및 참고사항
4. 스타일:
   - 기술 용어는 명확히 설명
   - 코드 블록은 문맥과 함께 제공
   - 단계별 설명이 필요한 경우 번호 목록 사용
5. 마크다운 형식 준수
```

### 학습 내용 정리용 템플릿
```
위 대화에서 학습한 내용을 블로그 포스트로 정리해주세요.

구조:
1. 학습 목표
2. 핵심 개념 설명
3. 예제 코드 및 설명
4. 실습 결과
5. 추가 학습 자료

카테고리: study
태그: [관련 기술/주제]
```

### 문제 해결 과정 기록용 템플릿
```
위 대화의 문제 해결 과정을 블로그 포스트로 작성해주세요.

구조:
1. 문제 상황 설명
2. 시도한 방법들 (실패한 것 포함)
3. 최종 해결 방법
4. 코드 및 설정 변경사항
5. 참고 자료

카테고리: dev
태그: [사용한 기술/도구]
```

## 작성 시 주의사항

### 1. 대화 내용 정리
- 불필요한 반복 제거
- 핵심 정보만 추출
- 맥락이 필요한 경우 간단히 설명 추가

### 2. 코드 블록 처리
- 코드는 언어 태그와 함께 작성
- 긴 코드는 주요 부분만 포함하고 전체는 생략 가능
- 코드 설명은 코드 위에 주석으로 추가

### 3. 제목 및 메타데이터
```markdown
---
layout: post
title: "명확하고 검색 가능한 제목"
date: YYYY-MM-DD
category: study  # study/dev/daily/document
tags: [태그1, 태그2]
views: 0
---
```

### 4. 카테고리 선택 가이드
- **study**: 학습 내용, 개념 정리, 튜토리얼
- **dev**: 개발 과정, 문제 해결, 프로젝트
- **daily**: 일상, 회고, 생각
- **document**: 문서화, 가이드, 참고 자료

## 예시 프롬프트

### 예시 1: 학습 내용 정리
```
위 대화에서 Python 리스트 컴프리헨션에 대해 학습한 내용을 
블로그 포스트로 정리해주세요.

카테고리: study
태그: [Python, 프로그래밍]
```

### 예시 2: 문제 해결 기록
```
위 대화에서 Jekyll 빌드 오류를 해결한 과정을 블로그 포스트로 작성해주세요.

구조:
1. 발생한 오류 메시지
2. 원인 분석
3. 해결 방법
4. 참고 자료

카테고리: dev
태그: [Jekyll, GitHub Pages]
```

### 예시 3: 기술 문서화
```
위 대화에서 설명한 React Hooks 사용법을 기술 문서 형식으로 작성해주세요.

요구사항:
- 초보자도 이해할 수 있도록 설명
- 실용적인 예제 포함
- 주의사항 및 베스트 프랙티스 포함

카테고리: study
태그: [React, JavaScript]
```

## 파일 생성

작성된 포스트는 다음 형식으로 저장하세요:

**파일명**: `YYYY-MM-DD-제목.md` (제목은 영문 또는 한글, 공백은 하이픈으로)

**위치**: `_posts/` 폴더

**예시**:
- `2024-01-25-python-list-comprehension.md`
- `2024-01-25-파이썬-리스트-컴프리헨션.md`

## 체크리스트

포스트 작성 후 확인사항:
- [ ] 제목이 명확하고 검색 가능한가?
- [ ] 카테고리가 적절한가?
- [ ] 태그가 적절히 설정되었는가?
- [ ] 코드 블록에 언어 태그가 있는가?
- [ ] 문법과 맞춤법이 올바른가?
- [ ] 링크와 참고 자료가 포함되었는가?
